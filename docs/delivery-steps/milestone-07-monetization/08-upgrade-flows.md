# Upgrade & Downgrade Flows

**Objective:** Implement user flows for upgrading and downgrading subscription tiers.

**Tasks:**

- [ ] Add UI for upgrade/downgrade actions
- [ ] Handle proration and billing changes
- [ ] Confirm changes with user
- [ ] Update access and limits on change
- [ ] Test all upgrade/downgrade scenarios

**Acceptance Criteria:**

- Users can upgrade or downgrade tiers
- Proration and billing changes are handled
- Access and limits update immediately
- All flows are tested
